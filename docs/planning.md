# 기획서 — 친구 스케줄 매칭 앱

> 작성일: 2026-05-12 | 작성자: 1인 팀

## 1. 비전

바쁜 일상 속 친구들과 약속 잡기 어려운 사람을 위해, 각자의 고정/일회성 스케줄을 등록해두면 날짜별로 누가 되는지 한눈에 확인하고 초대·수락까지 이어지는 모바일 앱

## 2. 핵심 플로우

```
[설정] 고정 스케줄 등록 (매주 반복되는 일정)
          +
     일회성 스케줄 추가 (특정 날에만 있는 일정)
          ↓
[메인 캘린더] 날짜 선택
          ↓
[친구 상태 확인] 해당 날짜에 일정 있는 친구 / 없는 친구 구분 표시
          ↓
[초대] 만나고 싶은 친구 선별 → 초대 발송 (푸시 알림)
          ↓
[초대 응답] 수락 / 거절(사유) / 보류(사유)
          → 거절·보류 시 사유가 초대자에게 알림으로 전달
          ↓
[채팅방] 수락한 사람끼리 자동으로 채팅방 생성
```

## 3. 사용자 시나리오

**시나리오 1: 고정 스케줄 등록 후 약속 잡기**
직장인 A는 앱 설정에서 매주 월·수·금 저녁 스케줄을 고정 등록해둔다. 주말에 친구들과 만나고 싶어 캘린더에서 토요일을 선택하자, 친구들 중 누가 되는지 / 안 되는지 한눈에 표시된다. A는 되는 친구들을 선별해 초대를 보내고, 수락한 친구들끼리 채팅방이 자동으로 만들어진다.

**시나리오 2: 일회성 스케줄 등록 + 거절 사유 수신**
대학생 B는 이번 주 금요일 시험이 잡혀 일회성 스케줄을 추가한다. 친구 C가 금요일 모임 초대를 보내자, B는 거절하며 "시험이 있어서"라는 사유를 입력한다. C에게는 B의 사유가 알림으로 전달된다.

**시나리오 3: 수락자끼리 채팅방 생성**
D가 5명의 친구에게 초대를 보냈고, 3명이 수락·2명이 거절했다. 수락한 3명과 D, 총 4명이 포함된 채팅방이 자동으로 생성되어 세부 장소와 시간을 이야기한다.

## 4. MoSCoW 분류

### Must (없으면 출시 불가)
- 이메일/비밀번호 회원가입 및 로그인
- 친구 추가 (검색 및 요청/수락)
- **고정 스케줄 등록** (요일 반복 설정)
- **일회성 스케줄 등록** (특정 날짜)
- **캘린더 UI** — 날짜별 본인 일정 표시
- **날짜 선택 시 친구 가용 상태 조회** (일정 있음 / 없음)
- **초대 선별 및 발송** (푸시 알림)
- **초대 응답** — 수락 / 거절(사유) / 보류(사유)
- **사유 알림** — 거절·보류 시 초대자에게 푸시 알림
- **수락자 간 채팅방 자동 생성**

### Should (없어도 데모 가능)
- 채팅방 내 메시지 기능 (텍스트 채팅)

### Could (시간이 남으면)
- 초대 보류 후 재응답 기능
- 채팅방 내 이미지 공유
- 약속 D-day 리마인더 알림

### Won't (이번에는 하지 않음 — 범위 폭주 방지)
- 지도 연동
- SNS 공유
