# 위험 체크리스트 — 친구 스케줄 매칭 앱

> 작성일: 2026-05-12

---

## 위험: 다른 프로젝트 병행으로 인한 시간 부족
- 카테고리: 일정
- 발생 가능성: 상
- 영향도: 상
- 대응:
  - 예방: 주간 작업 시간을 미리 블로킹하고, WBS 일정을 보수적으로 설정
  - 발생 시: Should 기능(채팅) 과감히 제거하고 Must에 집중
- 모니터링 지표: 주차별 Must 기능 완료율

## 위험: 겹치는 시간 찾기 알고리즘 복잡도
- 카테고리: 기술
- 발생 가능성: 중
- 영향도: 상
- 대응:
  - 예방: 알고리즘을 먼저 종이에 설계하고, 단위 테스트로 검증
  - 발생 시: 단순화 (겹치는 날짜만 표시 → 시간대까지 순차 확장)
- 모니터링 지표: 3주차 내 알고리즘 동작 여부

## 위험: React Native / Firebase 처음 사용
- 카테고리: 기술
- 발생 가능성: 중
- 영향도: 중
- 대응:
  - 예방: 2주차에 환경 세팅 + 간단한 프로토타입으로 스파이크 진행
  - 발생 시: 공식 문서 및 AI Agent 적극 활용, 커뮤니티 예제 코드 참고
- 모니터링 지표: 2주차 말 로그인 화면 동작 여부

## 위험: Firebase 무료 티어 한도 초과
- 카테고리: 기술
- 발생 가능성: 하
- 영향도: 중
- 대응:
  - 예방: Firestore 읽기/쓰기 최소화, 불필요한 실시간 리스너 제거
  - 발생 시: 읽기 캐싱 추가, 테스트 계정 수 제한
- 모니터링 지표: Firebase 콘솔 사용량 주간 확인

## 위험: 데모 시점에 앱이 동작하지 않음
- 카테고리: 결과물
- 발생 가능성: 중
- 영향도: 상
- 대응:
  - 예방: 발표 3일 전(6/6) 최종 빌드 완료 및 전체 시나리오 테스트
  - 발생 시: 사전 녹화 영상으로 대체
- 모니터링 지표: 6/6 완성도 점검 체크리스트 통과 여부

## 위험: AI가 만든 코드를 이해 못해 Q&A 실패
- 카테고리: AI 의존
- 발생 가능성: 중
- 영향도: 상
- 대응:
  - 예방: AI가 작성한 코드는 반드시 직접 읽고, 핵심 로직에 주석 추가
  - 발생 시: ADR 문서를 화면에 띄우며 결정 근거 설명
- 모니터링 지표: 코드 리뷰 자기 체크 (매 기능 완료 시)

## 주간 점검 체크리스트

- [ ] 다른 프로젝트 일정과 충돌이 없는가?
- [ ] Firebase 사용량 한도에 여유가 있는가?
- [ ] AI가 작성한 코드를 직접 이해하고 설명할 수 있는가?
