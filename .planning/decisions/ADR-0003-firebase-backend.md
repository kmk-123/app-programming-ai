# ADR-0003: 백엔드 — Firebase 선택

- 상태: Accepted
- 날짜: 2026-05-12
- 결정자: 1인 팀

## 배경

스케줄 데이터 저장, 사용자 인증, 푸시 알림, 실시간 채팅(Should)을 위한 백엔드를 선택해야 한다. 1인 팀으로 서버 운영 부담 없이 빠르게 개발해야 한다.

## 고려한 대안

### 대안 A: Firebase
- 장점: 인증/DB/알림/채팅 모두 한 플랫폼에서 해결, 무료 티어 충분, React Native SDK 공식 지원, 실시간 데이터 동기화 기본 제공
- 단점: 복잡한 쿼리 제한, 무료 한도 초과 시 과금, Firebase 종속성

### 대안 B: Supabase
- 장점: SQL 기반으로 복잡한 쿼리 가능, 오픈소스
- 단점: 실시간 채팅 구현 추가 작업 필요, Firebase보다 React Native 레퍼런스 적음

### 대안 C: 직접 서버 (Node.js + DB)
- 장점: 완전한 커스터마이징 가능
- 단점: 서버 구축/운영 비용과 시간 부담, 1인 팀에 비현실적

## 결정

**Firebase** (Firestore + Auth + Cloud Messaging + Realtime Database)를 선택한다.

## 이유

- 인증, 데이터 저장, 푸시 알림, 채팅을 하나의 플랫폼에서 처리 가능
- 서버 운영 없이 빠른 개발 가능 (1인 팀 적합)
- React Native와의 공식 통합이 잘 되어 있음

## 결과 (예상되는 영향)

긍정:
- 백엔드 인프라 관리 불필요, 개발 속도 향상
- 실시간 데이터 동기화로 겹치는 일정 표시 용이

부정 / 제약:
- Firebase 무료 티어 읽기/쓰기 한도 주의 필요
- 복잡한 일정 비교 쿼리는 클라이언트 사이드에서 처리해야 할 수 있음

## 후속 작업

- [ ] Firebase 콘솔에서 프로젝트 생성
- [ ] Firestore 보안 규칙 설계
- [ ] 월간 사용량 한도 모니터링 설정
